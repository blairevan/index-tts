# IndexTTS2 命令行工具 - 音频停顿、连读与自定义发音配置文档

本文档介绍如何在 IndexTTS2 命令行工具中实现并使用**音频增加停顿**、**连读（连续）**以及**自定义发音（多音字/生僻字）**功能。

---

## 一、 功能概述

在进行数字人播报或长音频合成时，为了提升语音的自然度与准确性，系统通过支持**特定英文文本标记（Markup）**来允许：
1. **音频增加停顿**：通过在文本中插入 `[pause:秒数]` 标签，可实现任意位置的精确无声停顿。
2. **标记连读（连续）**：通过在文本中使用 `[connect:文字]` 标签，显式指明这部分文本在语音合成时需要连续发音，不插入任何语气停顿。
3. **自定义发音设置**：通过在文本中使用 `[汉字|拼音+声调]` 标签，显式指定某些字（如多音字、异体字）的正确读音。

---

## 二、 文本标记格式规范

### 1. 音频增加停顿
* **标签格式**：`[pause:时间秒数]`
* **时间范围**：支持浮点数或整数，例如 `0.5`、`1.5`、`2` 等，单位为秒。
* **使用示例**：
  ```text
  欢迎使用数字人系统，[pause:1.2]请选择您的播报音色。
  ```
  *在“系统，”和“请选择”之间会产生一个 1.2 秒的无声静音期。*

### 2. 标记连读（连续）
* **标签格式**：`[connect:连续读的字词]`
* **使用场景**：当两个词或词群之间不需要任何默认的气流停顿时，用连读标签包裹。在底层合成时，会剥离外部标签并将文字合并传入，保证自回归生成时的连贯性。
* **使用示例**：
  ```text
  我们今天[connect:哈哈哈哈]舍去这些烦恼。
  ```
  *“哈哈哈哈”会被紧密地连读。*

### 3. 自定义发音设置
* **标签格式**：`[目标汉字|拼音声调数字]`
* **声调规范**：拼音后附加声调数字（1：阴平/第一声，2：阳平/第二声，3：上声/第三声，4：去声/第四声，5：轻声）。对于拼音中的 `ü` 符号，一律用英文字母 `v` 代替（例如 `lv4`、`nve4`）。
* **使用示例**：
  * 多音字：
    ```text
    他性格很[舍|she4]得，但这次却在宿舍里犯了难。
    ```
    *第一个“舍”字将强制读作第四声（shè）。*
  * 生僻字/技术词：
    ```text
    这款车性价比极高，主打[率|lv4]性与动力。
    ```
    *“率”强制读作第四声（lǜ）。*

## 三、 命令行 (CLI) 调用指南

修改后的 `cli_v2.py` 工具原生支持上述所有标记，并在生成 SRT 格式字幕时，自动剔除这些标记，还原为干净的字词并确保时间轴对齐。

### 运行命令示例
```bash
python cli_v2.py \
  --text "大家好，[pause:1.5] 我们今天[connect:哈哈哈哈][舍|she4]去烦恼。" \
  --voice "examples/sample_prompt.wav" \
  --output "outputs/output_demo.wav" \
  --srt "outputs/output_demo.srt" \
  --json_subtitle "outputs/output_demo.json"
```

### 运行输出说明
1. **音频文件 (`outputs/output_demo.wav`)**：
   * 合成“大家好，”
   * 插入 `1.5` 秒静音
   * 合成“我们今天哈哈哈哈舍去烦恼。”（其中“哈哈哈哈”强制连读，“舍”字强制发音为 `she4`）
   * 自动拼接成一段完整的高质量音频。
2. **字幕文件**：
   * 生成的 SRT/JSON 字幕中将不包含停顿、连读和拼音标签，文字部分会干净地显示为 `"大家好， 我们今天哈哈哈哈舍去烦恼。"`，且静音段内不会显示字幕。

## 四、 前端对接建议

由于前端在另一个项目中，可参考以下交互与传输设计：

### 1. 前端编辑框输入规范
* **停顿插入**：在编辑框中插入“停顿按钮”，用户点击后弹窗选择时间，前端将标记 `[pause:1.0]` 插入至光标位置。
* **标记连读**：用户用鼠标框选想要连读的文本（如“哈哈哈哈”），点击“连读”按钮，前端将选中的文字转换为 `[connect:哈哈哈哈]` 插入编辑器。
* **发音设置**：用户选中特定文本，点击“发音设置”，弹窗录入拼音（如 `she4`），点击确定后，前端将文本格式化为 `[舍|she4]` 结构。

### 2. API 接口提交
前端向后端发起请求时，不需要预先解析这些标签，直接将带有 `[pause:xxx]`、`[connect:xxx]` 和 `[汉字|拼音]` 标签的**原始字符串**作为 `text` 参数发送给后端，由后端的 Python 脚本完成预处理、切分、多段合成及拼接。
